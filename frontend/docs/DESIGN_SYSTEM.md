# BreakPoint Design System

## Overview

This document defines the official visual identity of BreakPoint.

All UI components should follow this design system to maintain consistency throughout the application.

---

# Brand Personality

BreakPoint should feel:

- Professional
- Modern
- Premium
- Minimal
- Developer-focused
- Fast
- Confident
- Clean
- Spacious

Avoid making the interface look like a school project or a generic coding website.

---

# Shadows

Use soft shadows only.

Panels

Soft elevation

Buttons

Small shadow

Cards

Medium shadow

Avoid strong glowing effects.

---

# Spacing System

Use an 8px spacing scale.

Examples

- 8px
- 16px
- 24px
- 32px
- 40px
- 48px

Avoid random spacing values whenever possible.

# Cards

Style

- Rounded corners
- Soft border
- Comfortable padding
- Soft shadow

Cards should appear elevated from the background.

---

# Panels

Panels include:

- Sidebar
- Problem Panel
- Workspace
- Console

Requirements

- Rounded corners
- Consistent padding
- Soft border
- Soft shadow

---

# Inputs

Text fields

Dropdowns

Selectors

Requirements

- Rounded corners
- Clear focus state
- Comfortable padding

---

# Icons

Use simple outline icons.

Icons should be consistent throughout the application.

Do not mix icon styles.

---

# Animations

Animations should be subtle.

Allowed

- Hover transitions
- Color transitions
- Fade-ins
- Button press animation

Avoid

- Bouncing
- Flashing
- Excessive movement

---

# Workspace

The workspace is the heart of BreakPoint.

The coding experience should resemble a professional IDE while maintaining BreakPoint's unique identity.

The workspace should always contain:

- Problem Panel
- Monaco Editor
- Console
- Toolbar

---

# Responsive Design

Desktop First

Maintain a professional desktop experience.

Mobile

- Stack layouts vertically
- Avoid horizontal scrolling
- Preserve usability

---

# UI Inspiration

Use these products only as inspiration.

- VS Code
- Cursor
- GitHub
- Linear
- Vercel Dashboard

Do NOT copy their appearance.

BreakPoint must have its own identity.

---

# Design Principles

Every UI decision should follow these principles:

1. Consistency
2. Simplicity
3. Readability
4. Accessibility
5. Performance
6. Developer-first experience

Whenever a new component is created, it must follow this design system.
